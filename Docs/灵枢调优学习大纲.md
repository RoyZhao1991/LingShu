# 灵枢调优学习大纲

> 用途:以「一个回合的生命周期」9 个阶段为索引,逐站理解 LLM/agent 领域知识 + 找优化点。
> 工作法(每一站都遵守):**假设 → 量基线 → 只改一个变量 → 再量 → 没动就回退**;事实分两类标注:**①已知事实**(代码/数据坐实) vs **②经验性猜测**。出错可被纠正,证据为准。

## 读法
- **[新]** = 要学的 LLM/agent 领域知识(重点投放)　**[架]** = 你已会的架构本能(对齐术语即可)
- **⭐** = 这一站的核心概念(必拿下)　**🔴** = 优化大杠杆

## 重点投放
真正花时间的是 **⑤、⑥(大杠杆)+ ④(已确诊的真 bug)**;其余每站抓那 1-2 个 ⭐ 概念即可,别平均用力。

---

## ① 输入　·　杠杆:小　·　代码落点 `submitTextInput`
- [架] **串行输入队列**:为什么 agent 一次只处理一个回合(防上下文污染、防「未来问题」串台)
- [架] **多模态归一**:文本/附件/语音 → 统一成「文本 + 附件」
- [新] **「回合(turn)」是基本单位**:LLM agent 的世界是一回合一回合推进的

## ② 确定性快路径　·　杠杆:中(已做得好)　·　代码落点 内置技能拦截 / 声明式调用
- [新]⭐ **确定性 vs 概率性的分界**:哪些请求**根本不该喂模型**(本机直答/显式@/录制重放)
- [架] **声明式调用** `@agent·能力` = 确定性直达,跳过大脑误判
- [新] 这是延迟/可靠性的**隐形大杠杆**——codex 体验一大半来自「少问模型」

## ③ 分诊 Triage　·　杠杆:中　·　代码落点 `LingShuState+Triage`
- [新]⭐ **用 LLM 做分类(classification)**:意图识别、直答 vs 派发
- [新] **structured output**:让模型吐**可解析的结构**(枚举/JSON),不是自由文本
- [新] **小而快的分类 prompt**:单轮、无工具
- [架] 分诊本身也是一次模型调用——简单请求上**值不值**

## ④ 目标认知 + 能力核验　·　杠杆:中(真 bug 在这)　·　代码落点 `GoalSpec` / `deriveGapAnalysis` / 能力台账
- [新]⭐ **结构化目标(GoalSpec)**:模糊请求 → objective + 成功标准 + 约束;为什么是「可验收/可控」的地基
- [新]⭐ **能力注册表/台账**:确定性事实(我有什么能力)**vs** LLM 猜 —— 病根
- [新] **能力沉淀与复用(procedural memory)**:获取一次 → 记住 → 下次复用
- [新] **1-shot 调用模式**:无工具、单轮、专做一个认知判断
- [架] 「**概率组件回答确定性问题**」的味道 —— 你的架构嗅觉,这站正好练

## ⑤ 上下文装配　🔴　·　杠杆:大　·　代码落点 mainAgentSession / HarnessConfig / agentBuiltinTools
- [新]⭐ **token / 分词**:文本怎么变 token,为什么 17K「贵」
- [新]⭐ **context window**:定长窗口 = 稀缺资源,一切围绕它省
- [新]⭐ **system prompt**:agent 真正的「源码」,行为写在这
- [新]⭐ **tool schema 成本**:21 个工具每回合占多少 token
- [新]⭐ **prompt caching(前缀缓存)**:暖会话为什么便宜、冷启动为什么贵 —— 直对你的实测
- [新] **lean vs thick**:按需注入 vs 全量灌
- [架] 这站优化 = 「把每回合的 payload 变小/可缓存」,像减小每次 RPC 的包

## ⑥ Agent 循环　🔴　·　杠杆:大　·　代码落点 `runLoop` / ModelGateway
- [新]⭐ **agent loop / ReAct**:理解→调工具→看结果→再来,这就是 agent 的心跳
- [新]⭐ **tool use / function calling**:模型怎么「调用」工具(它只吐个结构,**执行是你做的**)
- [新]⭐ **每回合成本结构**:输入是**全量重发**的 → 总成本 ≈ 往返次数 × 上下文大小
- [新]⭐ **streaming**:token 流式,为什么「感觉快」
- [新] **parallel tool calls**:一回合多个无依赖工具并发,砍往返
- [新] **max_tokens / 输出预算**:早先那个 4096 截断就是这
- [新] **模型差异**:DeepSeek 工具调用为什么不稳、Claude 为什么稳 —— **模型是个旋钮**
- [架] **循环不变量 / 终止条件**:防 wedge、防死循环 —— 你的主场

## ⑦ 上下文压缩　·　杠杆:中　·　代码落点 compactHistoryIfNeeded
- [架] **窗口溢出 = 缓存淘汰问题**:留什么、扔什么、怎么扔 —— 你的主场
- [新] **有损 vs 近无损**:纯摘要(有损)vs 摘要 + 可检索细节(知识图谱,近无损)
- [新] **分层保留**:系统/seed 永留 + 最近 N 条逐字 + 中段蒸馏

## ⑧ 验收门　·　杠杆:中　·　代码落点 DeliveryReview / CompletionGate
- [新]⭐ **确定性门 vs LLM-as-judge**:能用测试/运行判的,别用模型自评
- [新] **古德哈特定律**:指标一旦变成死命优化的目标,就不再反映真实
- [新] **maker ≠ checker**:独立会话验收(防自己夸自己)
- [新] **假阳/假阴**:刚修的「假缺口找你要授权」就是假阳
- [架] 验收 = 契约校验,只是对象变成了概率输出

## ⑨ 产出 + 记忆　·　杠杆:小　·　代码落点 task record / dreaming / 知识图谱
- [新]⭐ **LLM 没有内生记忆**:每次调用失忆,「记忆」全靠**外部检索再注入**
- [新] **短期(上下文)vs 长期(知识图谱/向量库)**
- [新] **RAG(检索增强)** 基本盘:存 → 检索 → 注入
- [新] **记忆巩固(consolidation)**:把回合经验提炼进长期记忆
