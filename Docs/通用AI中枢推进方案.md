# 灵枢 → 通用 AI 中枢:架构差距评估与推进方案

> 交给后续线程的**自包含**落地方案。读完即可动手。
> **铁律(继承现有文化):通用零定制(领域只进数据/知识,绝不进核心分支)· 安全红线只增不减 · 每步真测 + .app E2E + soak · 真假分级(✅/⚠️/❌)零假 demo · 模块化(纯逻辑抽函数配单测,State 扩展 ≤500 行)。**
> 配套阅读:`Docs/架构速查手册.md`(组件地图/现状)、`Docs/灵枢内核ABI.md`(五协议固核)、`Docs/对标前沿Agent_harness追平方案.md`(harness 追平)。

## 0. 目标(冷启动必读)

灵枢的最终形态**不是"会很多功能",而是**:用户提出**目标**,灵枢在合法/安全/授权/资源允许下,**理解目标 → 评估能力 → 发现缺口 → 补齐能力 → 调度资源 → 执行 → 验收结果 → 沉淀经验 → 持续迭代自己**。核心面向所有能力/资源/目标,**不写死任何领域**(PPT/代码/炒菜机/机器人都不进核心)。

最终验收标准:① 用户提目标,灵枢能判断目标→评估能力→发现缺口→补齐→规划→调度→执行→监控→验收→沉淀;② 超出当前能力的目标**不简单拒绝**,而是说明缺口+规划补齐路径+在授权内尝试扩展;③ 遇到未知软件/设备/服务/模型/agent,能主动识别/学习/接入;④ 接入新能力/设备/系统**不要求改核心**。

---

## 1. 现状诊断:下半身壮、上半身缺(独立验收实测,非文档自述)

**执行 / 具身 / 外围自生长(下半身)出乎意料地完整**;**认知控制平面(上半身)基本空白**。灵枢目前把用户请求**直接当指令丢进 agent 循环即兴串执行**,中间没有"先把目标想清楚→评估能不能做→缺什么→怎么补→做完真验收→沉淀"的执行脑。这正是"会执行任务的 agent → 通用中枢"的分水岭。

## 2. 15 条要求逐条评分(基于真实代码)

| # | 要求 | 评 | 现状(代码实证) |
|---|---|---|---|

| 10 | 长期任务 | ✅ | 嵌套分阶段循环 + orchestrator + DAG + watch_until + schedule(持久化)+ 暂停恢复 + 撞顶恢复 |

| 12 | 记忆经验 | ⚠️ | 知识图谱 + dreaming 蒸馏 skill + 本机知识;**未把"目标→计划→缺口→失败→成功路径→新能力→验收"作为闭环沉淀+下次接续** |
| 13 | 自我迭代 | ❌ | `author_component` **只改外围插件**(设计上不碰核心);可当编码 agent 改自己仓库,但**无"发现不足→改核心→测→回滚/发布"自治闭环** |
| 14 | 通用性 | ✅ | 内核 ABI 显式固化"核心不依赖领域、外围插件化、不重编译";契约测试守门 |

## 3. 推进路线(认知控制平面,按依赖自底向上)

> 每项:现状 / 方案(通用) / 改哪些文件 / 验收 / 超越点。**全部建在现有内核+工具之上,不重造下半身。**

### P1 · GoalSpec(目标认知)— 基石,#1
- **现状**:请求直接进循环,无结构化目标。
- **方案**:`submitTextInput`/`runMainAgentTurn` 入口先产一份结构化 `LingShuGoalSpec`(objective/constraints/boundaries/risks/success_criteria/open_questions/kind=task·interaction·question)。模型产(零领域分支),解析容错,可澄清,持久化进任务记录;全循环(规划/验收/记忆)引用它。
- **改哪些文件**:新增 `Sources/Support/LingShuGoalSpec.swift`(纯类型+`LingShuGoalSpecParser` 解析+解析提示)、`Sources/State/LingShuState+GoalSpec.swift`(派生+落痕)、入口轻接(`runMainAgentTurn`)。
- **验收**:`GoalSpecTests`(解析 干净JSON/围栏/夹叙/缺字段/垃圾→nil/kind 判定);.app live 提一个含约束+风险的目标,trace 见结构化 GoalSpec。
- **落地分步**:**P1a 骨架(本期):纯类型+解析+单测 + 入口观察态接入(产出并落痕,绝不改执行)。P1b:循环/验收/记忆开始引用它(进 P2/P3)。**

### P2 · 能力图谱 + 缺口分析(前置)— #2#3#4 升级
- **现状**:`list_capabilities` 是平面枚举;只有反应式升级,无前置缺口分析。
- **方案**:把枚举升级成可查 `CapabilityGraph`(我会什么/谁能做/怎么调/成本/风险/在线否/怎么验);`GapAnalyzer` 据 GoalSpec×图谱判缺(模型/工具/设备/权限/知识/资源/资金/人类确认),缺则规划补齐(复用 `author_component`/`discover_*`/`acquire_resource`)。让"超能力目标不拒绝,而是说明缺口+规划补齐+授权内尝试扩展"成真。
- **改哪些文件**:升级 `LingShuCapabilityProvider`/新增 `LingShuGapAnalyzer`(纯逻辑);接进入口(GoalSpec 后、执行前)。
- **验收**:纯逻辑单测(给定 GoalSpec+能力集→正确判缺+补齐路径);.app live 提一个缺能力的目标,灵枢说明缺口+给补齐方案而非空答。

### P3 · 通用验收按成功标准 — #11
- **现状**:OutcomeVerification 二元(动作/文档)。
- **方案**:泛化为**按 GoalSpec.success_criteria 的多类型验收**(文件/代码测试/设备效果/环境变化/用户确认),验"目标达成"而非"文件存在"。可插拔注册表(复用 `LingShuArtifactVerifierRegistry` 思路)。
- **改哪些文件**:扩 `LingShuOutcomeVerification`/`LingShuArtifactVerifier`、接 `LingShuState+DeliveryReview`/`+AgentAcceptance`。
- **验收**:单测(各成功标准类型→正确判定);.app live 动作型目标"写文档冒充"判不过、真效果判过。

### P4 · 经验闭环 — #12
- **现状**:图谱+dreaming 有,但 GoalSpec 生命周期未沉淀。
- **方案**:每个目标的(计划/缺口/失败原因/成功路径/新增能力/验收结果)结构化沉淀进图谱;下次同类目标在**规划阶段**接续(不从零)。
- **改哪些文件**:接 `Sources/Memory/`、dreaming 钩子、GoalSpec 生命周期落库。
- **验收**:单测(同类目标召回历史经验);.app live 第二次同类目标更快/复用上次路径。

### P5 · 强/中/弱脑分层 + 降级 — #9
- **现状**:单脑 + 自适应 harness;无脑分层路由/降级。
- **方案**:把自适应 harness + 本地确定性提升为**显式脑分层路由**——弱脑(本地规则/缓存/轻量:权限拦截/设备心跳/急停/简单问答)、中脑(常规规划/拆解/调度)、强脑(复杂目标理解/缺口分析/自我扩展/重大决策);**强脑不可用→降级运行不瘫痪**。
- **改哪些文件**:`LingShuModelChannel`/新增脑路由器、复用 `HarnessProfile`/`LocalIntentResolver`/`HeartbeatPolicy`。
- **验收**:单测(路由判定+强脑掉线降级);.app live 断强脑仍能弱脑应答/拦截/心跳。

### P6 · 自我进化闭环 — #13(最高风险,最后)
- **现状**:author_component 只改外围;无核心自迭代闭环。
- **方案**:检测自身不足(失败验收/遥测/不变量)→提改造方案→改核心(强门控)→测→soak→回滚/发布。靠现有回归/soak 闸兜底。
- **改哪些文件**:新增 `LingShuSelfEvolutionLoop`;复用 fuzz/soak/不变量/git。
- **验收**:单测(检测→提案→门控);严格 .app + soak,任何不过即回滚。

### 并行支线 · 真实硬件验证 — #6#7 落地
M3/M4(设备发现/执行器)是"机制已通、真设备待验"。有真外设(ESP32/舵机/继电器/智能插座)时按**现机制**实测闭环(不是再造)。

## 4. 执行策略与约束

- **顺序**:P1→P2→P3 是主链(目标→能力→验收的认知回路);P4 经验闭环可在 P3 后并入;P5 脑分层与主链正交、可并行;P6 最后。硬件支线随真设备到位。
- **每步**:纯逻辑抽函数配单测;`bash Scripts/build-app.sh debug` + MCP 8917 真机 E2E;接进 `Scripts/soak-e2e.sh` 守不回归;真假分级标注。
- **不变红线**:通用零定制(领域只进数据/知识)· 安全只增不减(越接近物理/不可逆/对外越严:风险判断+权限+身份确认+过程记录+急停)· 内核 ABI 形状变要升版本过契约测试。
- **改完同步** `Docs/架构速查手册.md`(组件地图/缺口)。

## 5. 一句话总纲

**灵枢已有强壮的"下半身"(执行/具身/外围自生长),缺的是"上半身"那层执行脑。补齐 `GoalSpec → 能力图谱/缺口分析 → 通用验收 → 经验闭环 → 强弱脑 → 自我进化`,且全部建在现有内核+工具之上、领域只进数据、安全红线只增不减——灵枢就从"会执行任务的 AI" 变成"能评估自己、扩展自己、调度世界资源完成目标的通用中枢"。**
