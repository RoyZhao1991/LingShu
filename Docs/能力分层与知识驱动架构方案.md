# 灵枢:能力分层(大脑优先·框架兜底)+ 知识驱动 —— 落地方案

> 交给执行线程的自包含方案。读完即可动手。**铁律:全程通用、零定制;陈述非祈使;安全是硬闸不分层。**

---

## 0. 背景(冷启动必读)

灵枢 = 大脑(大模型推理)+ 四肢(工具)+ 感知 + 壳(本仓库)。核心范式已是「统一 agent 循环」(`Sources/LingShuAgentLoop.swift`,内核 ABI 见 `Docs/灵枢内核ABI.md`)。

**已踩到的根本矛盾(本方案要解决的):** 我们为了让(当前较弱的)大脑可靠,堆了一堆**程序性脚手架/框架**(`discover_devices`/`peripherals`/设备分类/已接入子串匹配/各种关键词清单),结果:① 对足够聪明的大脑是**天花板**(逼它走我的路子);② 自身脆、互相矛盾(设备"凭空消失"、写文档冒充接入)。

**拍板的方向(用户):**
1. **大脑能自己搞定 → 不加约束**,给薄原语 + 知识,让它自由发挥。
2. **大脑搞不定 → 才用通用框架兜底**协助。
3. 给大脑的应是**陈述性"决策知识"**(它推理时参考、自己拍板),不是**祈使性框架**(替它定步骤)。
4. **绝不要定制化**:任何内容必须通用;关键词清单/写死清单这类是定制,要撤。

## 1. 核心原则

**分五层,各司其职:**

| 层 | 是什么 | 对大脑 | 处置 |
|---|---|---|---|
| **原语** | 能力(run_command/网络/读写/`author_component`/感知/可控记忆) | 赋能 | 重投、做厚 |
| **知识** | 陈述性事实/教训(参考,自行判断) | 赋能 | 新建(知识图谱) |
| **框架** | 程序性脚手架/专用工具/步骤 | 约束(天花板) | **降为"失败兜底",不默认强加** |
| **安全** | 红线硬闸(不可逆确认/沙箱/不静默执行未审码) | 故意约束 | **永远硬闸,不分层、不因"脑能搞定"放松** |
| **呈现** | 可点击确认、TTS 念概要 | 无关 | 保留,与能力分层正交 |

**一句话:厚的是原语+知识+大脑,薄的是框架(只兜底),硬的是安全,旁路的是呈现。**

**旋钮性质:** 框架(脚手架)介入频率与脑力成反比。强脑几乎不触发兜底;弱脑(当前 DeepSeek)常触发。机制不变,随脑增减。

## 2. 机制:能力分层 = 失败升级的"脚手架阶梯"

对**每一个"既能让大脑自由做、又存在结构化兜底"的能力**,统一成一个升级阶梯。**关键:默认从最薄开始,只有确定性校验失败/停滞才升一级加脚手架。**

```
Rung 0(最薄):大脑 + 原语 + 召回的相关知识 + 极简提示  ← 默认从这开始
   │  verifyOutcome(goal) 确定性后置校验?  ── 过 → 完成
   │  不过 / 停滞 / 自述卡住 ↓
Rung 1(加引导):同上 + 一段结构化提示/清单(框架作为"建议"注入,仍是大脑做)
   │  verify? ── 过 → 完成 ; 不过 ↓
Rung 2(最确定):确定性工具/过程(不需大脑判断的那部分,存在才有)
   │  verify? ── 过 → 完成 ; 不过 → 诚实交还 + ask_user
```

- **`verifyOutcome(goal)` = 确定性后置条件校验(通用、maker≠checker)**:校的是**真实世界结果**,不是"大脑声称完成"。例:接入设备→"该设备现在真能控吗(试调一次拿到合法响应)";查资料→"拿到结构化结果了吗"。**绝不以"产出了文件/说了完成"判过**(根治"写指南文档冒充接入")。
- **复用现成**:`Sources/State/LingShuState+AgentAcceptance.swift` 的 `verifyAndContinue`(已有:撞顶恢复 + 多轮验收 + 停滞检测 + 预算)。**改造点**:把"验收不过 → 原样重试大脑"升级为"**验收不过 → 升一级脚手架再试**"(注入更强引导 / 切到确定性兜底)。
- **触发升级的三信号**(任一):`verifyOutcome` 失败 / 停滞检测命中(`stuckRepeatThreshold`/只读空转) / 大脑 `ask_user` 自述缺信息。
- **适用范围**:只对"有可校验结果 + 有结构化兜底"的能力(设备发现/接入/控制、查询解析、文件定位等)。纯生成型(PPT/写作)无确定性兜底,**仍走现有验收**,不强套本阶梯。

**实现建议**:抽一个通用编排 `runCapabilityWithEscalation(goal, verify, rungs:[guidance])`(纯逻辑部分可单测:给定 verify 结果序列,断言升级路径)。挂在现有 `verifyAndContinue` 旁或并入。

## 3. 知识库:决策知识(陈述性),不是框架(祈使性)

**底座:复用记忆 v2 知识图谱**(`Sources/Memory/`:`LingShuMemoryNote`/`LingShuMemoryGraph`/`LingShuMemoryGardener`/`LingShuKnowledgeGraph`;接线 `Sources/State/LingShuState+KnowledgeGraphDreaming.swift`)。**不要为设备/家居另建 KB**——它已是通用的原子笔记 + 语义召回 + dreaming 园丁自维护机制。

**判据(区分知识 vs 框架,入库时必须守):**
- ✅ 知识(陈述句·留):"CozyLife 灯走局域网 TCP 5555,JSON+`\r\n`,`cmd:3 {attr:[1],data:{1:1}}` 开/`0` 关,DP3 亮度 0-1000,DP4 色温" / "HomeKit 配件单控制者独占,已进苹果家庭的第三方连不了" / "教训:对'接入设备'类请求,写指南文档≠真接入"。
- ❌ 框架(祈使句·禁):"接入设备时先 X 再 Y"。

**三件事:**
1. **获取(边做边学)**:大脑探到一个事实(probe 协议 / web_search / 撞到教训)就 `remember` 入图谱;dreaming `consolidateKnowledgeGraph` 蒸馏会话为**陈述性事实+教训**(调其提示:"只抽事实和教训的陈述句,禁止抽操作步骤")。由大脑自决记什么(通用 `remember`,非写死例程)。
2. **召回(作参考)**:决策前按相关性注入,措辞固定为 **「相关知识·供参考,自行判断是否适用」**(现有 additive 召回即此,`recallText`)。**绝不写成规则/必须**。
3. **纪律(红线)**:入库校验"陈述非祈使"(园丁/remember 落库前过一道:命中祈使/步骤特征则拒或改写);召回措辞永远"参考"。**这道纪律是"知识不退化成框架"的保证,必须有单测。**

## 5. 安全(硬闸,不参与"大脑优先",不可被知识推翻)

- 这些**永远是确定性硬闸**,不因"大脑足够聪明"放松,也**绝不能做成"知识"**(否则脑一聪明就推理掉了):
  - 自编/未审来源代码绝不静默执行(静态门 `LingShuSkillSafetyGate` + P3 沙箱 `LingShuPluginSandbox` + LLM 风险审)。
  - 不可逆/对外动作执行前确认(`requestShellApproval`;执行器物理动作每次确认)。
- 见 [[skill-self-evolution]] / `Docs/灵枢内核ABI.md` §3。**改造中这层一行不动、只增不减。**

## 10. 硬约束(铁律,违反即回退)

1. **通用、零定制**:不写关键词清单/设备特判/品牌分支;领域特定的东西(如某设备协议)只能是**知识笔记(数据)或自编组件(数据)**,绝不进壳代码。
2. **陈述非祈使**:给大脑的知识是事实/教训,不是步骤;框架只当失败兜底,不默认强加。
3. **安全硬闸**:红线层不分层、不可被知识推翻、只增不减。
4. **真凭据 + 真假分级(✅/⚠️/❌),零容忍假 demo**,交付附实测证据。
5. **以 .app 运行**(`open dist/灵枢.app`),改源码必 `bash Scripts/build-app.sh debug` 重建;MCP 8917 验证;日志 `/tmp/lingshu-control.log`。
6. **模块化**:纯逻辑抽函数配单测;Swift 文件 ≤500 行(状态扩展)/ ≤800(基础设施),架构守卫 `ArchitectureGuardTests` 守住;改完同步更新 `Docs/架构速查手册.md`。
7. 危险/不可逆/对外动作先问;做不到的真机步骤如实标"待人工验"。

## 11. 一句话总纲

**默认让大脑用薄原语 + 召回的陈述性知识自己干;确定性校验真实结果,失败才升级到通用框架兜底;安全永远硬闸,呈现层正交;所有领域特定只存在于"知识/自编组件"这类数据里,壳代码零定制。脚手架是随脑力可调的旋钮,内核原语与安全红线不动。**
