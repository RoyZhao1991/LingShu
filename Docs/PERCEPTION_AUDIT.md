# 多模态感知深度审计：哪些是真的，哪些还是演示级

> 2026-06-12。逐文件审读 `Sources/Services/`（语音/视觉/感知网关/认主）后的真实性结论。
> 配套工具：感知面板的「感知自检」按钮，八项链路逐一实测（权限、采样、首帧延迟、合成产出、云端可达），
> 结论以实测为准。

## 一、真实可用的部分（实测可验证）

### 语音输入链
- **麦克风采集**：`AVAudioEngine` 真实 tap，RMS 电平实时计算（驱动极简模式波形的是真电平，不是动画）。
- **Apple Speech 流式识别**：`SFSpeechRecognizer(zh_CN)` 增量转写，支持本机识别时强制 on-device，带标点。
- **内嵌 SenseVoice ASR**：`LingShuEmbeddedASRRuntimeLocator` 对 sherpa-onnx 运行时做真实文件探测
  （runtime / model.onnx / tokens.txt / silero_vad.onnx），缺失时真实回退 Apple Speech 并如实提示——
  注意：这是"运行时已就绪才可用"的可选增强，机器上没装 sherpa-onnx 时它不假装存在。
- **连续通话 VAD**：`LingShuVoiceCallController` 80ms tick 状态机——说话阈值 0.12 起句、
  静音 0.06 持续 1s 收口自动提交、应答完自动续听。是真实信号驱动，不是定时器演戏。
- **打断（barge-in）**：灵枢播报中用户电平超 0.2 持续 0.25s → 停 TTS、立即重听。真实现。

### 视觉链
- **摄像头采集**：`AVCaptureSession` 真实取流，1fps 节流分析（节流是刻意的功耗权衡）。
- **本地视觉解析**：Vision 框架真实推理——`VNDetectFaceLandmarksRequest` 人脸关键点、
  `VNRecognizeTextRequest` 中英 OCR（fast 级）、亮度采样、JPEG 关键帧打包（0.42 质量）。

### 感知网关与云路由
- **信号封包与转发**：转写/视觉摘要/音频块/视频帧封成统一 envelope，按 provider 节流（原始流 0.8s/次）真实转发；
  云端失败自动回退本地解析并如实标注状态。
- **云感知客户端**：真实 HTTP 调用（models 列表 / 图像分析），自检里测的是真实往返延迟。

## 一点五、云端场景理解链路（2026-06-12 实测）

客户端已全链路接通：视觉请求开启 `include_qwen_semantics`，提示词明确要求"场景描述+人数与
人物状态"；回包的场景语义置于态势摘要首位；用户开口时按需刷新最新帧的场景理解（仅远端路由，
本地解析模式不出网）。

**服务端现状（实测）**：数据网关的 OCR 与目标检测正常（0.3s 往返），但场景语义后端异常——
返回 `qwen_semantics_failed: 400 Client Error ... http://127.0.0.1:8104/v1/chat/completions`，
即网关侧的语义模型服务（127.0.0.1:8104）配置或部署有问题。**需要网关运维侧修复**；客户端
就绪，服务端恢复后场景理解自动生效。此异常现在会浮出在态势摘要和感知自检（云感知项报"降级"），
不再静默吞掉。

## 二、演示级/局限（不要当成已交付能力）

1. **认主身份锁是轻量特征锁**：面容特征 = 人脸关键点采样坐标（每区域取 3 点），声线特征同样简化。
   防误唤醒可以，**防有意冒充不行**。照片攻击、相似声线都可能通过。架构文档已把它定位为
   "可演示的轻量特征锁，等待 H100/云端声纹/人脸识别替换特征提取层"——这个定位是诚实的，照此使用。
2. **"运动检测"是亮度差**：帧间平均亮度变化，不是光流/物体跟踪。挥手和开灯它分不清。
3. **视觉摘要是规则拼接**：「光线正常，画面稳定，检测到 1 张人脸」来自阈值规则，不是模型描述。
   真正的场景理解依赖云感知路由（接通时真实可用）。
4. **端到端延迟无全链路埋点**：自检能测单项（首帧、合成、云往返），但"麦克风停顿→首句播报"
   的全链路 1.5s 目标还没有 signpost 埋点（Phase 4）。
5. **TTS 分句流式未做**：回复整段合成后播报，长回复首响慢（Phase 4）。

## 三、隐私边界提示（与感知数据零留存约束相关）

- 云感知路由转发的是**原始 JPEG 帧 / PCM 音频块**。一旦转发，数据离开本机；
  "云端零留存"依赖服务端承诺，本地无法技术验证。对此敏感时，把感知路由切回「本地解析」，
  所有分析在机内完成，不出网。
- 认主档案（`~/Library/Application Support/LingShu/Identity/owner-profile.json`）只存特征向量与阈值，
  不存原始音频/视频帧。已核实代码与文档一致。

## 四、自检项与判定标准

| 自检项 | 实测方式 | 通过标准 |
| --- | --- | --- |
| 麦克风采集 | 独立引擎采样 1.2s | 权限 authorized 且峰值电平 > 0.0001 |
| Apple Speech | 权限 + zh_CN 识别器状态 | authorized 且 isAvailable |
| 内嵌 ASR | sherpa-onnx 文件探测 | 全部文件就绪（缺失=降级，Apple Speech 兜底） |
| 语音合成 | 离线渲染「自检」到缓冲 | 3s 内产出音频帧 |
| 摄像头采集 | 实测首帧 | 3.5s 内产出 JPEG 帧（报分辨率/体积/延迟） |
| 视觉解析 | 最近观测新鲜度 | 10s 内有真实观测 |
| 云感知路由 | 真实调 models 接口 | 返回模型列表（报往返延迟） |
| 认主身份锁 | 档案与样本计数 | 开启且两类样本齐备（永远附"轻量锁"提示） |
